### Пять требований для Dart кода

1. Явное приведение типов при работе с dynamic
   - Всегда приводить к конкретному типу перед вызовом методов
   - `(value as int?)?.compareTo()` вместо `value.compareTo()`
   - Использовать `as Type?` с null-safety для безопасного приведения

2. Null-safety обработка
   - Использовать `??` для значений по умолчанию
   - Использовать `?.` для цепочек вызовов на nullable объектах
   - Проверять на null перед использованием или использовать `!` только когда 100% уверен

3. Типизация коллекций
   - `List<Map<String, dynamic>>` вместо `List<dynamic>`
   - `Map<String, dynamic>` вместо `Map`
   - При создании из dynamic использовать `.cast<Type>()` или явное приведение элементов

4. Возвращаемые типы функций и замыканий
   - Явно указывать тип возврата для sort/map/where если компилятор не выводит
   - `list.map<Widget>((item) => ...)` вместо `list.map((item) => ...)`
   - Для compareTo результат должен быть `int`, не `dynamic`

5. Константы и final
   - `const` для compile-time констант (виджеты без параметров, значения)
   - `final` для runtime значений которые не меняются
   - `static const` для констант класса
